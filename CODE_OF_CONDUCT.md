# Code of Conduct

## Our Pledge

We want Command Shelf to be an open, respectful, and constructive project for everyone involved.

## Our Standards

Examples of behavior that contributes to a positive environment include:

- showing respect for differing viewpoints and experience levels
- giving and receiving feedback constructively
- assuming good intent while being direct about technical concerns
- focusing discussion on the work rather than the person

Examples of unacceptable behavior include:

- harassment or personal attacks
- discriminatory language or conduct
- deliberate intimidation
- publishing private information without permission
- persistent bad-faith disruption of discussion or review

## Enforcement

Project maintainers are responsible for clarifying and enforcing these standards and may remove, edit, or reject comments, commits, code, wiki edits, issues, and other contributions that violate this Code of Conduct.

## Scope

This Code of Conduct applies in project spaces and in public spaces when someone is representing the project.

## Reporting

If you experience or witness unacceptable behavior, report it to the project maintainers through the repository contact channels.
